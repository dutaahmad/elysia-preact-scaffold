## Quick Start

```bash
# Install dependencies
bun install

# Start development (FE + API concurrently)
bun run dev
```

- **Frontend**: http://localhost:5173
- **API**: http://localhost:3000

The Vite dev server proxies `/api/*` requests to the Elysia backend automatically.

## Commands

| Command | Description |
|---------|-------------|
| `bun run dev` | Start both Vite frontend (:5173) and Elysia API (:3000) with watch mode |
| `bun run dev:fe` | Start only the Vite frontend |
| `bun run dev:be` | Start only the Elysia API with watch mode |
| `bun run build` | Build the frontend for production (outputs to `dist/`) |
| `bun run preview` | Run production server (serves `dist/` + API on :3000) |
| `bun run db:generate` | Generate SQL migration files with Drizzle Kit |
| `bun run db:migrate` | Push schema changes to the SQLite database |
| `bun run db:studio` | Open Drizzle Studio (database GUI) |
| `prelysia feat <name>` | Generate a new CRUD module (server + client) |
| `prelysia feat <name> --fe-only` | Generate only frontend assets for a module |
| `prelysia remove <name>` | Remove a module and its database table |

## Adding Modules

Use the `prelysia` CLI to generate full CRUD modules in seconds:

```bash
prelysia feat categories
```

This interactive command prompts for field definitions and generates:

- **Server**: schema, types, model, service, routes, plugin (6 files)
- **Client**: types, API client, List page, Form page
- **App wiring**: Registers routes, sidebar link, and imports

Each generated module follows the same pattern — consistent, predictable, and easy to customize.

## Tech Stack Details

### Database

SQLite is embedded via Bun's native `bun:sqlite` driver with Drizzle ORM. No external database server required. The database file lives at `server/data/app.db` (configurable via `DB_PATH`).

- Schema changes: `bun run db:migrate`
- Migration files: `bun run db:generate`

### Frontend

Preact provides a React-compatible API at a fraction of the bundle size. The app uses `wouter` for client-side routing and `@tanstack/preact-query` for server state management.

### Theme

Dark/light theme is toggled via `.ThemeToggle` and persisted to `localStorage`. The theme is applied before first paint via an inline script in `index.html` to prevent flash.

## Deployment

1. Build the frontend:
   ```bash
   bun run build
   ```

2. Start the production server:
   ```bash
   bun run preview
   ```

   This serves the built `dist/` folder via `@elysia/static` with `indexHTML: true` for SPA routing, while keeping API routes live on the same port.

For production, set `NODE_ENV=production` and ensure `PORT` is configured for your hosting environment.
